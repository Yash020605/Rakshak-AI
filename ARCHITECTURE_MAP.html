🖥️ Backend (Node.js + Express)
🚀 server.js — Entry Point
Express + HTTP + Socket.IO server.

Port 5000 CORS *

• Connects to MongoDB via Mongoose
• Mounts all API routes under /api
• Serves /dashboard (static HTML)
• Serves /uploads (media files)
• Socket.IO rooms: responder-dashboard, user:{id}
🚨 routes/alert.js
POST /sos POST /sms GET /:id PATCH /:id/resolve PATCH /:id/dispatch

Dual-Routing on SOS:
→ Family Route: SMS via Twilio to all contacts
→ Responder Route: find nearby responders (5km) + Socket.IO notify
→ Background: find nearest service via Overpass API
→ Dispatch: notifies victim + family + dashboard
📍 routes/location.js
POST /update GET /live/:userId GET /nearest GET /route

• Live location stored in node-cache (in-memory)
• Nearest service via Overpass API (OSM)
• Driving route via OSRM public API
• Returns GeoJSON coordinates + ETA minutes
👤 routes/profile.js

⚡ SOS Alert Flow — Step by Step

📱 User Triggers SOS

1
User taps Medical / Fire / Police button on HomeScreen
2
App vibrates, gets GPS coordinates (pre-fetched for speed)
3
alertService.js checks network (NetInfo)
4
Online: POST /api/alert/sos with userId, type, lat, lng
5
Offline fallback: Direct SMS via react-native-send-intent
6
Navigate to MapScreen with alertId + nearestService

🖥️ Backend Processes SOS (Dual-Routing)

1
Validate request (express-validator)
2
Fetch user Profile from MongoDB (name, blood group, contacts)
3
Create Alert document in MongoDB (status: active)
4
Cache live location in node-cache
5
Respond immediately to mobile (alertId returned)
6
Route A: Twilio SMS to all emergency contacts
7
Route B: Find responders within 5km → Socket.IO notify dashboard
8
Background: Overpass API → nearest hospital/station → emit to mobile

📊 Responder Dashboard Receives Alert

1
Socket.IO new-alert event received in browser
2
Alert card appears in sidebar, map marker added, alert sound plays
3
Browser notification shown (if permission granted)
4
Responder clicks alert → modal shows patient photo, blood group, prescriptions
5
Responder clicks "Accept & Dispatch" → enters name/unit
6
PATCH /api/alert/:id/dispatch → alert status → dispatched

🚑 Live Tracking After Dispatch

1
Backend emits alert-dispatched:{userId} to victim's Socket.IO room
2
Mobile MapScreen shows "Help is on the way" banner
3
Dashboard starts browser geolocation watch (GPS)
4
Every ~3s: responder GPS → OSRM route API → draw polyline on map
5
Socket.IO responder-location event → relayed to victim's room
6
ETA displayed on dashboard modal (distance + minutes)
7
Victim taps "Resolved" → PATCH /api/alert/:id/resolve → status: resolved

🔌 Socket.IO Real-Time Events
↑ CLIENT EMITS
join-responder-dashboard
Dashboard joins responder room. Updates responder socketId in DB.
↑ CLIENT EMITS
join-user-channel
Mobile app joins personal room user:{userId} to receive dispatch updates.
↑ CLIENT EMITS
responder-location
Dashboard streams GPS {alertId, userId, lat, lng, eta} → relayed to victim's room.
↓ SERVER EMITS
new-alert
Broadcast to responder-dashboard room when SOS is triggered.
↓ SERVER EMITS
alert-dispatched:{userId}
Sent to victim's room when responder accepts. Mobile shows "Help is on the way".
↓ SERVER EMITS
alert-dispatched:{alertId}
Also emitted by alertId so MapScreen can listen by alertId.
↓ SERVER EMITS
responder-location:{alertId}
Relayed to victim's room. Mobile MapScreen shows moving responder marker.
↓ SERVER EMITS
nearest:{userId}
Sent after Overpass API resolves nearest hospital/station. Mobile updates map.
